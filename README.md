# 🦆 Duck Revolution - A Chaotic 2D Action Game

**A goofy, chaotic 2D action game built in Unity where ducks rise up against humanity!**

*Inspired by the tone, pacing, and destructibility of Broforce.*

## 🎮 Game Concept

One day, ducks gain consciousness and decide humans have oppressed them long enough. The ducks go completely berserk and start attacking humanity. The humans respond with escalating forces: police officers, SWAT teams, and eventually the military.

**THE DUCK REVOLUTION HAS BEGUN!**

## 🚀 Getting Started

### Step 1 — Install Unity 6

1. Download and install [Unity Hub](https://unity.com/download)
2. In Unity Hub → **Installs** → **Install Editor** → find **Unity 6** (6000.x LTS)
3. No extra modules needed beyond the base editor

### Step 2 — Open the Project

1. In Unity Hub → **Projects** → **Open** → select this repository's root folder (the one containing `Assets/`, `ProjectSettings/`, `Packages/`)
2. Unity will import all packages and compile scripts — **this takes a few minutes on first open**

> ⚠️ **You will see an empty grey scene when the project first opens. This is expected.** The game scene is generated by the setup tool below.

### Step 3 — Import TextMeshPro Essentials

Unity will likely show a popup: *"TMP Essentials"* — click **"Import TMP Essentials"**. The game's UI won't display correctly without it.

If the popup doesn't appear automatically:
1. Go to **Window → TextMeshPro → Import TMP Essential Resources**
2. Click **Import**

### Step 4 — Run the Setup Tool

1. Wait until Unity finishes compiling — the bottom-right corner of the editor will stop showing a spinning progress indicator
2. Check the **Console** window (**Window → General → Console**) — there should be **no red errors**
3. In the Unity **menu bar** at the very top of the screen, click **DuckRevolution → Setup Game (Full Auto Setup)**

> If the **DuckRevolution** menu is missing, see [Troubleshooting](#-troubleshooting) below.

The setup tool will automatically:
- Generate all placeholder sprites (duck, police, SWAT, army, projectiles, etc.)
- Create ScriptableObject enemy configurations
- Build all prefabs (player, enemies, weapons, props)
- Construct the full game scene — ground, platforms, UI, camera, spawn points

A confirmation dialog will appear when it's done.

### Step 5 — Play!

Press the **▶ Play** button at the top of the Unity Editor.

**THE DUCK REVOLUTION BEGINS!**

## 🌊 Wave System

| Wave | Enemies | Description |
|------|---------|-------------|
| 1–2 | Police Officers | Basic enemies with pistols |
| 3–4 | SWAT + Police | Tougher enemies with burst fire |
| 5+ | Army + SWAT + Police | Soldiers with grenades |

Enemies get progressively harder and more numerous each wave.

## 🦆 Duck Abilities

- **Feather Shot** — Rapid-fire feather projectiles
- **Egg Grenade** — Bouncing explosive eggs with area-of-effect damage
- **QUACK** — Stuns all nearby enemies (5-second cooldown)

## 🐛 Troubleshooting

**"DuckRevolution" menu doesn't appear in the menu bar**
- The Editor script only registers the menu once all C# scripts compile without errors
- Open **Window → General → Console** and fix any **red errors** first
- If there are no errors, try **Assets → Refresh** (Ctrl/Cmd + R) to force a recompile

**"TMP Essentials" popup appears — what do I do?**
- Click **"Import TMP Essentials"** — this is required for all UI text to display correctly

**I see an empty grey or 3D scene when the project opens**
- This is normal — the scene hasn't been generated yet
- Run **DuckRevolution → Setup Game (Full Auto Setup)** to create it (Step 4 above)

**Player falls through the ground**
- Make sure the Ground object has the layer set to **Ground**
- Make sure the Player's GroundCheck child object has the Ground layer mask configured

**Enemies don't spawn**
- Check that WaveManager has all three enemy prefabs assigned in the Inspector
- Confirm spawn point objects exist in the scene Hierarchy

**Nothing happens when pressing Play**
- Open **Window → General → Console** and check for errors
- Make sure `GameManager` exists in the scene Hierarchy

## 🔧 Customization

**Adding new enemy types**
1. Create a script inheriting from `EnemyBase`
2. Override `PerformAttack()` for custom behaviour
3. Create an `EnemyData` ScriptableObject with stats
4. Create a prefab and assign it to `WaveManager`

**Modifying waves**
- Edit `WaveManager.cs` fields: `baseEnemiesPerWave`, `enemiesPerWaveIncrease`
- Update `GetEnemyPrefabForWave()` to control enemy composition per wave

**Adding powerups**
- Add new weapon types in `WeaponSystem.cs`
- Create pickup scripts that modify `PlayerController` or `WeaponSystem` at runtime
